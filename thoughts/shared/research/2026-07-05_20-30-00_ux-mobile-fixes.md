# Research: UX Issues - Mobile Horizontal Scrolling and Tooltip Overlay Fixes

**Date**: 2026-07-05T20:30:00-07:00
**Researcher**: Claude
**Git Commit**: 9c72ebd
**Branch**: main
**Repository**: game

## Research Question

User reported UX issues including horizontal scrolling on mobile and weird overlays (screenshot showed FirstTimeHint tooltip appearing misaligned on Cheese Vat row in GeneratorPanel).

## Summary

Identified and fixed 8 distinct UX issues:

1. **FirstTimeHint tooltip positioning broken for left/right positions** - The component used `left-1/2 -translate-x-1/2` for all positions, breaking horizontal placement
2. **Z-index hierarchy violations** - Multiple components using wrong z-levels (tooltips at z-50 modal level)
3. **Horizontal overflow risk** - Missing `overflow-x: hidden` on html/body
4. **Undersized touch targets** - Several buttons below 44px minimum for mobile accessibility
5. **Fixed-width modals** - EquipmentModal used `min-w-[320px]` which could overflow narrow viewports
6. **EquipmentModal using wrong prop** - Passed z-index via `className` instead of `zIndexClass`
7. **DebugPanel could exceed viewport** - No max-width constraint
8. **Toast/notification overflow risk** - No max-width constraints on fixed-position elements

## Detailed Findings

### Issue 1: FirstTimeHint Tooltip Positioning (CRITICAL)

**File**: `src/components/ui/shared/FirstTimeHint.tsx:60`

The tooltip used the same horizontal centering for ALL positions:
```tsx
// Before - BROKEN for left/right
className={`absolute ${positionClasses[position]} left-1/2 -translate-x-1/2 z-50 w-64`}
```

**Fix**: Position classes now include axis-appropriate translations:
```tsx
const positionClasses = {
  top: 'bottom-full mb-2 left-1/2 -translate-x-1/2',
  bottom: 'top-full mt-2 left-1/2 -translate-x-1/2',
  left: 'right-full mr-2 top-1/2 -translate-y-1/2',
  right: 'left-full ml-2 top-1/2 -translate-y-1/2',
};
```

Also changed z-index from `z-50` (modal) to `z-30` (tooltip level per hierarchy).

### Issue 2: Z-Index Hierarchy Violations

Multiple components were using `z-50` (modal level) when they should be lower:

| Component | Before | After | Rationale |
|-----------|--------|-------|-----------|
| FirstTimeHint | z-50 | z-30 | Tooltips should be below modals |
| SaveToast | z-50 | z-40 | Floating UI level |
| GoldenCheeseNotification | z-50 | z-40 | Floating UI level |
| DebugPanel | z-50 | z-40 | Floating UI level |

**Files changed**:
- `src/components/ui/shared/FirstTimeHint.tsx:60`
- `src/components/ui/SaveToast.tsx:24`
- `src/components/ui/GoldenCheeseNotification.tsx:229`
- `src/components/ui/DebugPanel.tsx:194`

### Issue 3: Horizontal Overflow Prevention

**File**: `src/index.css:140-148`

Added `overflow-x: hidden` to both html and body to prevent any horizontal scroll:
```css
body {
  overflow-x: hidden;
}
html {
  overflow-x: hidden;
}
```

### Issue 4: Touch Target Accessibility

Multiple buttons were below the 44px minimum for mobile accessibility:

| Component | Element | Before | After |
|-----------|---------|--------|-------|
| PartyFormationPanel | Remove hero button | w-5 h-5 (20px) | w-11 h-11 (44px hitbox, 20px visual) |
| AudioControls | Mute button | p-2 (~36px) | min-w-[44px] min-h-[44px] |
| FirstTimeHint | Dismiss X | text only | min-w-[44px] min-h-[44px] |
| FirstTimeHint | Got it button | py-1 | py-2 min-h-[44px] |
| EquipmentModal | Close button | w-8 h-8 (32px) | min-w-[44px] min-h-[44px] |

### Issue 5: EquipmentModal Fixed Width

**File**: `src/components/ui/EquipmentModal.tsx:220`

Removed `min-w-[320px]` which could cause horizontal overflow when combined with modal padding (352px total required):
```tsx
// Before
<div className="... min-w-[320px] ...">

// After  
<div className="... max-w-md w-full ...">
```

### Issue 6: EquipmentModal zIndexClass Prop

**File**: `src/components/ui/EquipmentModal.tsx:219`

The z-60 was passed via `className` instead of `zIndexClass`:
```tsx
// Before - WRONG
<ModalOverlay ... className="z-60">

// After - CORRECT
<ModalOverlay ... zIndexClass="z-60">
```

### Issue 7: DebugPanel Viewport Overflow

**File**: `src/components/ui/DebugPanel.tsx:194-195`

Added max-width constraint:
```tsx
// Before
<div className="fixed top-16 left-4 z-50">
  <div className="w-80 sm:w-96 ...">

// After
<div className="fixed top-16 left-4 z-40 max-w-[calc(100vw-2rem)]">
  <div className="w-80 sm:w-96 max-w-full ...">
```

### Issue 8: Toast Container Constraints

**File**: `src/components/ui/SaveToast.tsx:24`

Added max-width to prevent overflow:
```tsx
<div className="fixed bottom-4 right-4 z-40 flex flex-col gap-2 max-w-[calc(100vw-2rem)]">
```

## Architecture Insights

### Z-Index Hierarchy (Documented in index.css:103-121)

```
z-10 to z-20  : Internal element layering (decorative overlays, icons)
z-30          : Tooltips, hints (RennetDisplay, FirstTimeHint, version display)
z-40          : Floating UI (toasts, notifications, debug panel, feedback widget)
z-45          : Dropdowns that need to appear above toasts (AudioControls)
z-50          : Standard modals (OfflineProgressModal, AgingConfirmModal)
z-60          : Higher modals (EquipmentModal, CombatResultsModal)
z-70          : Priority modals (ChangelogModal, KeyboardHelpModal)
z-80          : Consent dialogs (PrivacyConsent)
z-90          : Loading screens (LoadingScreen)
z-100         : Critical overlays (BetaAgreement, SettingsPanel)
```

### Touch Target Pattern

For buttons that need a large touch target but small visual appearance:
```tsx
<button className="w-11 h-11 flex items-center justify-center">
  <span className="w-5 h-5 bg-red-500 rounded-full">×</span>
</button>
```

### Horizontal Scroll Prevention

Triple-layer approach:
1. `html { overflow-x: hidden }` - Global
2. `body { overflow-x: hidden }` - Body level
3. Layout component `overflow-x-hidden` - App wrapper
